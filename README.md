# Score + Kubernetes Experiment

This repo is a small working example of using [Score](https://score.dev/) with `score-k8s` to deploy a sample web app to a local Kubernetes cluster.

It includes:
- A Score workload (`score.yaml`) for a Node.js sample app
- Resource declarations for Postgres, DNS, and HTTP routing
- A generated Kubernetes manifest bundle (`manifests.yaml`)
- A bootstrap script (`deploy.sh`) that prepares a local cluster with Gateway API + NGINX Gateway Fabric

## Repo Layout

- `score.yaml`: Source workload spec (what the app needs)
- `manifests.yaml`: Rendered Kubernetes resources from `score-k8s generate`
- `gateway.yaml`: Shared `Gateway` named `default`
- `deploy.sh`: Local cluster and gateway setup script
- `.score-k8s/state.yaml`: `score-k8s` state for generated resource instances

## Prerequisites

Install these tools before running anything:
- `kind`
- `kubectl`
- `helm`
- `curl`
- `jq`
- `score-k8s`

## Quick Start

1. Create the local cluster and install gateway components:

```bash
bash deploy.sh
```

2. (Optional) Regenerate manifests from the Score spec:

```bash
score-k8s generate score.yaml -o manifests.yaml
```

3. Apply the application manifests:

```bash
kubectl apply -f manifests.yaml
```

4. Wait for the workload to become ready:

```bash
kubectl rollout status deploy/hello-world
```

## Verify

Get the generated route hostname:

```bash
kubectl get httproute -o jsonpath='{.items[0].spec.hostnames[0]}'
```

Then open `http://<hostname>/` in your browser (the setup maps traffic to local port `80`).

You can also inspect resources with:

```bash
kubectl get deploy,svc,httproute,statefulset,secret
```

## Notes

- The DNS host is generated by the default `dns` provisioner and is usually a `*.localhost` name.
- The Postgres resource is provisioned via the default `postgres` template provisioner in `score-k8s`.
- `manifests.yaml` contains generated resource names and values tied to the current state; regenerating can change them.
